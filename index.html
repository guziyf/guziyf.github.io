Objective-C概述

发表于 2021-04-01 分类于 iOS开发

  • OC的本质还是C语言

    • 在C的基础上新增了一小部分面向对象的语法
    • 将C的语法封装的更为简单
    • OC完全兼容C语言

  • m代表message，代表OC中最重要的一个机制-消息机制

  • main函数仍然是OC程序的入口和出口

    • int类型的返回值代表程序的结束状态
    • main函数的参数：仍然可以接受用户在运行程序的时候传递数据给程序
      • 参数也可以不要

  • #import

    • 预处理指令
    • 作用：是#include指令的增强版，将文件的内容在预编译的时候拷贝写指令的地方
    • 同一个文件无论import多少次，只会包含一次

  • 框架

    • 一个功能集，苹果或第三方将一些程序在开发程序的时候经常要用到的功能事先写好，把这些功能封装在一个类或者函数之中，这些函数和类的集合就叫框架
    • Foundation框架
      • 基本功能，输入输出、数据类型

  • @autoreleasepool{}自动释放池

    • 代码写在池中

  • NSLog(@”Hello, World!”);

    • printf函数的增强版

      • 输出一些调试相关的信息

      2021-04-01 16:13:32.939802+0800 Day01-OCbase[69990:6058063] Hello, World!

      • 会自动换行，如果最后加了\n，自动换行就会失效
      • OC中其实新增了一些数据类型

    • NSLog(@”语法”,变量列表);

  • 字符串

    • NSString类型指针变量，只能存储OC字符串地址
    • @”jack”，OC字符串前面必须要有一个@

  • NS前缀

    • NextStep —->Cocoa —->foundation框架中

  • OC中绝大部分关键字都是以@开头

  • 函数调用和C语言一样

  • 类的定义

    • @interface 类名 : NSobject
      • 这类事物具有的共同的特征，将他们定义为变量
    • @end
    • @implementation 类名
      • 将方法的实现写在这里
    • @end
    • 为类定义变量名时，前面要加下划线_
    • 如何创建一个类的对象
      • Person *p1 = [Person new];

一、概念认识

  • 神经网络：采集大量数据对构成数据集，把数据集喂入搭建好的神经网络结构，网络优化参数得到模型，模型读入新输入特征，输出识别结果。

  • 损失函数可以定量的判断W、b的优劣，当损失函数输出最小时，参数W、b会出现最优值。

    • 常用：均方误差
    • 目的：找一组参数w和b，使损失函数最小
    • 梯度下降法：沿损失函数梯度下降的方向，寻找损失函数的最小值，得到最优参数的方法。
    • 学习率：当学习率设置的过小时，收敛过程将变得十分缓慢。而当学习率设置的过大时，梯度可能会在最小值附近来回震荡，甚至可能无法收敛。
    • 反向传播：从后向前，逐层求损失函数对每层神经元参数的偏导数，迭代更新所有参数
      • 损失函数，对w的偏导是

二、预备知识

  • tf.where(条件语句，真返回A，假返回B)

  • np.random.RandomState.rand(维度)返回一个[0,1)之间的随机数

  • np.vstack(数组1，数组2)将两个数组按垂直方向叠加

  • np.mgrid

  • 神经网络(NN)复杂度

    • 空间复杂度

      • 层数=隐藏层的层数+1个输出层（注意区分节点和层）
      • 总参数=总w+总b

    • 时间复杂度

      • 乘加运算次数

    • 学习率

      • 指数衰减学习率，可以先用较大的学习率，快速得到较优解，然后逐步减小学习率，使模型在训练后期稳定。

  • 激活函数sigmoid()

    • 易造成梯度消失
    • 输出非0均值，收敛慢
    • 幂运算复杂，训练时间长
    • 还有Tanh函数、Relu函数、Leaky Relu函数

  • 损失函数(loss)：预测值(y)与已知答案(y_)的差距

    • 均方误差mse
      • loss_mse=tf.reduce_mean(tf.square(y_-y))

  • 交叉熵损失函数CE：两个概率分布之间的距离

    • softmax与交叉熵结合
      • tf.nn.softmax_cross_entropy_with_logits
      • 先符合概率分布，再求交叉熵

  • 缓解过拟合

    • 欠拟合
      • 增加输入特征项
      • 增加网络参数
      • 减少正则化参数
    • 过拟合
      • 数据清洗
      • 增大训练集
      • 采用正则化
      • 增大正则化参数
    • 正则化缓解过拟合：引入模型复杂度指标，利用给w加权值，弱化了训练数据的噪声。
      • L1正则化所有w求和，L2正则化所有w平方求和

  • 神经网络参数优化器

    • 待优化参数w，损失函数loss，学习率lr，每次迭代一个batch，t表示当前batch迭代的总次数
    • SGD，常用的梯度下降法
    • SGDM，增加一阶动量
    • Adagrad，增加二阶动量
    • RMSProp，SGD基础上增加二阶动量
    • Adam，同时结合SGDM一阶动量和RMSProp二阶动量

flex布局

发表于 2021-01-20 更新于 2021-01-21 分类于 weblearn

flex布局概念

  • 操作方便，布局极为简单，移动端应用很广泛
  • PC端支持差
  • IE11或更低版本不支持或仅部分支持

搭建HTML结构

flex布局原理

  • flexible Box弹性布局，任何一个容器都可以指定为flex布局
    • 当父盒子设为flex布局以后，子元素的float、clear和vertical-align属性将失效
    • 伸缩布局=弹性布局=伸缩盒布局=弹性盒布局=flex布局

flex布局父项常见属性

  • flex-direction:设置主轴的方向(row|row-reverse|column|column-reverse)
    • 默认主轴是x轴，侧轴是y轴
  • justify-content:设置主轴上的子元素排列方式
    • flex-start默认，从头部开始
    • flex-end从尾部开始
    • center在主轴居中对齐
    • space-around平分剩余空间
    • space-between先两边贴边 再平分剩余空间
  • flex-wrap:设置子元素是否换行
    • 默认不换行，装不开会缩小
    • nowrap|wrap

  • align-items:设置侧轴上的子元素排列方式（单行）

    • flex-start从上到下
    • flex-end从下到上
    • center一起居中
    • stretch拉伸

  • align-content:设置侧轴上的子元素的排列方式（多行）

    • 只能用于出现换行的情况，单行下没有效果
    • 比items多了space-between｜space-around

  • flex-flow:复合属性，相当于同时设置了flex-direction和flex-wrap

    • flex-flow: column wrap;

flex布局子项常见属性

  • flex属性

    • 定义项目剩余分配空间
    • .box{flex:1;}

  • align-self控制子项自己在侧轴上的排列方式

  • order控制排列顺序
    • 数值越小，排列越靠前，默认为0

定位

发表于 2021-01-18 更新于 2021-01-21 分类于 weblearn

为什么需要定位

  • 某个元素可以自由的在一个盒子内移动，并且压住其他盒子

  • 有些盒子固定在屏幕中的位置

  • 定位的组成

    • 定位=定位模式+边偏移

    • 定位模式用于指定一个元素在文档中的定位方式。边偏移则决定了该元素的最终位置

    • 定位模式通过position属性设置

      | 值 | 语义 |
      | ———— | ———— |
      | static | 静态定位 |
      | relative | 相对定位 |
      | absolute | 绝对定位 |
      | fixed | 固定定位 |

    • 边偏移有top、bottom、left和right4个属性，分别表示相对于父元素边线的距离

    • 静态定位static
      • 按照标准流特性摆放，没有边偏移
      • 很少在布局中使用
    • 相对定位relative
      • 相对于它原来的位置
      • 原来的位置在标准流中继续占有，不脱标
      • 最典型的应用是个给绝对定位当爹
    • 绝对定位absolute
      • 相对于祖先元素来说
      • 如果没有祖元素，或祖元素没有定位，则以浏览器定位
      • 如果祖元素有定位（相对、绝对、固定定位），则以最近一级的有定位祖先元素为参考点移动位置
    • 固定定位fixed
      • 固定在浏览器可视区
      • 跟父元素没有关系
      • 不能滚动条滚动
      • 还可以和版心始终贴在一起，让固定定位的盒子left: 50%. 走到浏览器可视区的一半位置；让固定定位的盒子margin-left版心宽度的一半距离，多走版心宽度的一半位置
    • 粘性定位sticky（了解，兼容性差）
      • 以可视窗口为参照点移动元素
      • 占有原先位置
      • 必需添加left\right\top\bottom中的一个才行

子绝父相

  • 子级绝对定位，不会占有位置，可以放到父盒子里面的任何一个地方，不会影响其他的兄弟盒子
  • 父盒子需要加定位限制盒子在父盒子内显示
  • 父盒子布局时，需要占有位置，因此父亲只能是相对定位

定位叠放次序

  • 如果属性相同，按照书写先后顺序，后面写的在上面
  • z - index数字越大盒子越靠上

定位的拓展

  • 绝对定位的盒子居中
    • 加了绝对定位的盒子不能通过margin: 0 auto水平居中
    • 水平居中：left:50% margin-left:-(盒子的一半)
    • 整体居中：再加一个top:50% margin-top:-(盒子的一半)
  • 定位特殊特性
    • 行内元素添加绝对或者固定定位，可以直接设置高度和宽度
    • 块级元素添加绝对或固定定位，如果不给宽度或者高度，默认大小是内容的大小
    • 脱标的盒子不会造成塌陷
  • 绝对定位和固定定位会完全压住盒子
    • 浮动的元素不会压住下面标准流的文字
      • 浮动之所以不会压住文字，因为浮动产生的目的最初是为了做文字环绕效果的，文字会围绕元素
    • 绝对定位或固定定位会压住

1.传统网页布局的三种方式

  • 普通流（标准流）、浮动、定位
    • 按照默认规定好的方式排列
    • 一个页面部分基本都包含了三种布局方式
  • 如何将多个块级放一排
    • 行内块
    • 浮动float
  • 网页布局第一准则：多个块级元素纵向排列找标准流，多个块级元素横向排列找浮动

2.什么是浮动

  • 用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘
  • 浮动元素会脱离标准流
    • 脱离标准流的控制（浮）移动到指定位置（动），即脱标
    • 浮动的盒子不再保留原先的位置，这个位置让其他标准流占用
    • 浮动的盒子应该在上层
    • 添加了任何浮动后，都会具有行内块元素的特性
  • 用标准流的父盒子约束

3.浮动布局注意点

  • 浮动和标准流的父盒子搭配

  • 一个元素浮动了，理论上其余兄弟元素也要浮动

  • 浮动的盒子只会影响浮动盒子后面的标准流，不会影响前面的

  • 盒子不一定需要定死高度，让孩子撑开

    • 清除浮动的本质

      • 清除浮动元素造成的影响

    • clear :left|right|both

    • 额外标签法（隔墙法）

      • 就是在最后一个元素后面加一个额外标签clear
      • 新增的盒子必须是块级元素，不能是行内元素

    • 父级添加overflow属性

      • 父元素中添加overflow:hidden
      • 缺点无法显示溢出的部分

    • 父级添加after伪元素（百度、淘宝、网易都这么用）

      • 给父元素加clearfix

    • 父级添加双伪元素
