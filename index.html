7.定位

发表于 2021-01-18 分类于 weblearn

为什么需要定位

  • 某个元素可以自由的在一个盒子内移动，并且压住其他盒子

  • 有些盒子固定在屏幕中的位置

  • 定位的组成

    • 定位=定位模式+边偏移

    • 定位模式用于指定一个元素在文档中的定位方式。边偏移则决定了该元素的最终位置

    • 定位模式通过position属性设置

      | 值 | 语义 |
      | ———— | ———— |
      | static | 静态定位 |
      | relative | 相对定位 |
      | absolute | 绝对定位 |
      | fixed | 固定定位 |

    • 边偏移有top、bottom、left和right4个属性，分别表示相对于父元素边线的距离

    • 静态定位static
      • 按照标准流特性摆放，没有边偏移
      • 很少在布局中使用
    • 相对定位relative
      • 相对于它原来的位置
      • 原来的位置在标准流中继续占有，不脱标
      • 最典型的应用是个给绝对定位当爹
    • 绝对定位absolute
      • 相对于祖先元素来说
      • 如果没有祖元素，或祖元素没有定位，则以浏览器定位
      • 如果祖元素有定位（相对、绝对、固定定位），则以最近一级的有定位祖先元素为参考点移动位置
    • 固定定位fixed
      • 固定在浏览器可视区
      • 跟父元素没有关系
      • 不能滚动条滚动
      • 还可以和版心始终贴在一起，让固定定位的盒子left: 50%. 走到浏览器可视区的一半位置；让固定定位的盒子margin-left版心宽度的一半距离，多走版心宽度的一半位置
    • 粘性定位sticky（了解，兼容性差）
      • 以可视窗口为参照点移动元素
      • 占有原先位置
      • 必需添加left\right\top\bottom中的一个才行

子绝父相

  • 子级绝对定位，不会占有位置，可以放到父盒子里面的任何一个地方，不会影响其他的兄弟盒子
  • 父盒子需要加定位限制盒子在父盒子内显示
  • 父盒子布局时，需要占有位置，因此父亲只能是相对定位

定位叠放次序

  • 如果属性相同，按照书写先后顺序，后面写的在上面
  • z - index数字越大盒子越靠上

定位的拓展

  • 绝对定位的盒子居中
    • 加了绝对定位的盒子不能通过margin: 0 auto水平居中
    • 水平居中：left:50% margin-left:-(盒子的一半)
    • 整体居中：再加一个top:50% margin-top:-(盒子的一半)
  • 定位特殊特性
    • 行内元素添加绝对或者固定定位，可以直接设置高度和宽度
    • 块级元素添加绝对或固定定位，如果不给宽度或者高度，默认大小是内容的大小
    • 脱标的盒子不会造成塌陷
  • 绝对定位和固定定位会完全压住盒子
    • 浮动的元素不会压住下面标准流的文字
      • 浮动之所以不会压住文字，因为浮动产生的目的最初是为了做文字环绕效果的，文字会围绕元素
    • 绝对定位或固定定位会压住

6.学成在线案例

发表于 2021-01-17 更新于 2021-01-18 分类于 weblearn

导航栏

  • 实际开发中，不会直接用链接a而是用li包含链接<li><a href=""></a></li>的做法

  • 如果直接用a，搜索引擎容易识别为有堆砌关键字嫌疑，影响网站排名

  • 给每个小a一个盒子，用display:block

  • 一般都要取消a的下划线text-decoration: none;，可以在最前面写
  • 当鼠标移动到a上盒子出现下边框a:hover{border-bottom: 2px solid #00A4FE;}
    • 注意别忘了设定线的样式
  • 用行高和盒子等高的方法使导航标签居中line-height: 42px;
  • 因为导航栏里面文字不一样多，所以最好给链接a左右padding撑开盒子，而不是指定宽度
  • 搜索框

banner栏

  • 背景透明度方法background-color: rgba(0, 0, 0, .3);
  • li取消前面的点list-style-type: none;

盒子上有一个超出盒子的“HOT”或“NEW”标签

  • 父亲如果没有定位，子级直接跑到整个界面的边缘上，超出盒子用负
    • 注意子绝父相

5.浮动

发表于 2021-01-16 分类于 weblearn

1.传统网页布局的三种方式

  • 普通流（标准流）、浮动、定位
    • 按照默认规定好的方式排列
    • 一个页面部分基本都包含了三种布局方式
  • 如何将多个块级放一排
    • 行内块
    • 浮动float
  • 网页布局第一准则：多个块级元素纵向排列找标准流，多个块级元素横向排列找浮动

2.什么是浮动

  • 用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘
  • 浮动元素会脱离标准流
    • 脱离标准流的控制（浮）移动到指定位置（动），即脱标
    • 浮动的盒子不再保留原先的位置，这个位置让其他标准流占用
    • 浮动的盒子应该在上层
    • 添加了任何浮动后，都会具有行内块元素的特性
  • 用标准流的父盒子约束

3.浮动布局注意点

  • 浮动和标准流的父盒子搭配

  • 一个元素浮动了，理论上其余兄弟元素也要浮动

  • 浮动的盒子只会影响浮动盒子后面的标准流，不会影响前面的

  • 盒子不一定需要定死高度，让孩子撑开

    • 清除浮动的本质

      • 清除浮动元素造成的影响

    • clear :left|right|both

    • 额外标签法（隔墙法）

      • 就是在最后一个元素后面加一个额外标签clear
      • 新增的盒子必须是块级元素，不能是行内元素

    • 父级添加overflow属性

      • 父元素中添加overflow:hidden
      • 缺点无法显示溢出的部分

    • 父级添加after伪元素（百度、淘宝、网易都这么用）

      • 给父元素加clearfix

    • 父级添加双伪元素
