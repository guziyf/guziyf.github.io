二、类和对象

1、面向对象（oop）

  • 对象是内存中专门用来存储数据的一块区域

  • 对象由三部分组成：

    • 标识（id）
    • 类型（type）
    • 值（value）

  • Python是一门面向对象的语言，Python中所有操作都是通过对象来进行的

  • 面向过程：通过对每个步骤的抽象，通过一个个小的步骤来完成一个功能，要实现别的功能，往往要重新编写代码，可复用性比较低。

    • 孩子上学
      • 妈妈起床
      • 妈妈叫孩子
      • 孩子吃早餐
      • 妈妈送孩子
      • 。。。。。。

  • 面向对象关注的是对象，而不是关注过程，步骤在对象里面，将所有功能统一保存到对应的对象中

    比如

    • 妈妈的功能保存到妈妈的对象中，孩子的功能保存到孩子对象中
    • 要使用每个功能，直接找对应的对象即可

  • 容易复用，编写稍微麻烦

  • 面向对象的思想：

    1. 找到对应的对象
    2. 让对象干事，实现功能

2、类的简介（class）

  • 目前所学习的对象都是Python内置好的对象

  • 但内置的并不能满足所有需求，所以我们在开发中经常需要自定义一些对象

  • 类，简单理解他就相当于图纸，在程序中我们需要根据类来创建对象

  • 类就是对象的图纸！

  • 也成对象是类的实例（instance）

  • 如果多个对象是通过一个类创建的，我们称这些对象是一类对象

  • 像int() float() bool() str() list()。。。这些都是类

  • a = int(10) # 创建一个int类的实例，等价于 a = 10

3、对象的创建流程

  • 类也是一个对象，用来创建对象的对象
  • 类的类型是type

  • 对象中的变量称为属性

  • 语法 ：对象，属性名 = 属性值

    mc.name = ‘孙悟空’

4、类的定义

  • 类和对象都是对现实生活中的实物或程序中的内容的抽象
  • 实际上所有的事物都由两部分构成：
    1. 数据（属性）
    2. 行为（方法）
  • 在类的代码，可以定义变量和函数，所有该类实例都可以通过对象.属性名访问
  • 方法调用时，会默认传递一个参数，所以定义一个方法时，至少要定义一个实参

5、属性和方法

  • 实例为什么能访问到类中的属性和方法
  • 当我们调用一个对象的属性时，解析器会先在当前对象中寻找是否含有该属性
  • 如果有，则直接返回当前的对象的属性值
  • 如果没有，则去当前对象的类对象中去寻找，如果有则返回类对象的属性值

  • 类对象和实例对象中都可以保存属性（方法）

    • 如果这个属性是所有实例共享的，则应该保存到类对象中
    • 如果这个属性是某个实例独有的，则应保存在实例对象中

  • 一般情况下，属性保存到实例对象中

  • 方法保存到类对象中

三、封装

1、封装简介

  • 面向对象三大特性之一：确保对象中的数据安全性
  • 隐藏对象中一些不希望被外部所访问的属性或方法
  • 如何获取（修改）对象中的属性
    • 需要提供一个getter和setter方法使外部可以访问到属性
    • getter获取对象中的指定属性（get_属性名）
  • 使用封装，确实增加了类定义的复杂程度，但也确保了数据的安全性
    • 隐藏了属性名，使调用者无法随意的修改对象中的属性
    • 设置方法控制属性是否只读
    • 在方法中可以增加数据验证，确保值是正确的
    • 或者进行数据记录

2、隐藏类中的属性

  • 双下划线开头的属性，是对象的隐藏属性，只能在类的内部访问，无法通过对象访问

  • 但实际上依然可以在外部访问

五、模块

1、模块化简介

  • 将一个完整的程序分解为一个一个小的模块
  • 采用模块化，将程序分别编写到多个文件中
  • 模块化优点：
    1. 方便开发
    2. 方便维护
    3. 模块可以复用

2、模块的创建

一个.py文件就是一个模块

  • import 模块名（python文件的名字不加py）
  • 可以引入同一个模块多次，但模块的实例只执行一次
  • 一般情况下import语句写在开头
  • Import 模块名 as 别名,给他起别的名字
  • Hello.py

  • Test_module.py

3、包（Package）

  • 包也是一个模块
  • 当我们模块中代码过多时，或者一个模块需要被分解为多个模块时，就需要使用包
  • 包是一个文件夹，里面必须要有_ _ int _ _ .py 文件
  • _ _ Pycache _ _ 是模块的缓存文件
  • py代码在执行前，需要被解析器先转换为机器吗，然后再执行
  • 所以使用模块（包）时，也需要将模块转换为机器码
  • 为了提高程序运行性能，python会在编译一次以后，将代码保存到一个缓存文件中，这样，下次加载这个模块（包）时，就可以不再重新编译而是直接加载缓存中编译好的代码即可
  • pyc编译过的文件

4、Python标准库

  • 为了实现开箱即用的思想，Python为我们提供了一个模块的标准库
  • sys模块，提供了一些变量和函数，可以获取到Python解析器的信息，或者通过函数来操作Python解析器
  • os模块

1、异常简介

  • 程序在运行的过程中，不可避免的出现一些错误，比如：

    • 使用了没有赋值的变量
    • 使用了不存在的索引
    • 除以0
    • 。。。。。。

  • 这些错误在程序中，称为异常

  • 这些异常会导致程序立即停止执行

2、异常传播
